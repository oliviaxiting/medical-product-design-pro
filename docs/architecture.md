# Medical Product Design Pro Architecture


## Project Philosophy

This project aims to build an AI-powered medical product design expert system.


The system contains three knowledge layers:


## 1. References

Purpose:

Provide external authoritative sources.

Examples:

- IEC62366
- ISO14971
- ANSI/AAMI HE75


References answer:

"What standards say?"


---


## 2. Knowledge

Purpose:

Convert standards into design principles.


Examples:

- human factors
- cognitive load
- safety design


Knowledge answers:

"What should designers understand?"


---


## 3. Rules

Purpose:

Provide AI decision logic.


Examples:

- Exposure control requires authorization separation.
- Critical alarms must not rely on color only.


Rules answer:

"What should AI recommend?"
