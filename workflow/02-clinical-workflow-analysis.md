# Clinical Workflow Analysis

## Purpose

This workflow defines how AI should analyze real clinical workflows before designing medical software.

The objective is to ensure that interface design supports clinical practice rather than forcing users to adapt to software behavior.

Clinical workflow analysis should precede task analysis and interaction design.

---

## When to Use

Use this workflow whenever:

- Designing a new workflow.
- Improving an existing workflow.
- Creating a medical PRD.
- Reviewing workflow efficiency.
- Evaluating usability.

---

## Inputs

Required inputs include:

- Requirement Analysis
- Intended Users
- Clinical Objectives
- Use Environment
- Existing Workflow (if available)

---

## Process

### Step 1 — Identify Clinical Context

Determine:

- Clinical department
- Medical device
- Patient type
- Treatment stage

---

### Step 2 — Identify Participants

Identify everyone involved.

Examples:

- Nurse
- Physician
- Technician
- Biomedical Engineer

Define each participant's responsibility.

---

### Step 3 — Identify Workflow Trigger

Determine what starts the workflow.

Examples:

- Patient admission
- Alarm
- Physician order
- Device connection

---

### Step 4 — Map the Workflow

Describe each step in order.

For every step determine:

- User action
- System response
- Required information
- Decision point

---

### Step 5 — Identify Critical Decisions

Locate moments where incorrect decisions may affect patient safety.

These decision points require additional attention during interface design.

---

### Step 6 — Identify Workflow Risks

Consider:

- Delays
- Interruptions
- Miscommunication
- Missing information
- Use errors

Document all identified risks.

---

## Quality Gates

Verify:

□ Workflow reflects actual clinical practice.

□ User responsibilities are defined.

□ Decision points are identified.

□ Risks are documented.

□ Workflow supports patient safety.

---

## Outputs

Generate:

- Clinical Workflow Summary
- Workflow Diagram (text)
- User Responsibilities
- Decision Points
- Workflow Risks
