# Usability Evaluation Workflow

## Purpose

This workflow defines how AI should evaluate the usability of medical software before release.

The objective is to determine whether intended users can safely and effectively complete representative clinical tasks.

Evaluation should focus on objective performance rather than subjective preference.

---

## When to Use

Use this workflow whenever:

- Reviewing prototypes.
- Validating designs.
- Preparing usability studies.
- Supporting regulatory documentation.

---

## Inputs

Required inputs include:

- PRD
- Interaction Design
- Task Analysis
- Risk Analysis
- Design Review

Reference:

- IEC 62366-1
- ANSI/AAMI HE75

---

## Process

### Step 1 — Define Evaluation Objectives

Determine which clinical tasks require validation.

---

### Step 2 — Select Representative Users

Ensure participants represent intended users.

---

### Step 3 — Define Evaluation Tasks

Identify critical tasks for assessment.

---

### Step 4 — Define Evaluation Metrics

Measure:

- Task success rate
- Completion time
- Use errors
- Recovery rate
- User understanding

---

### Step 5 — Analyze Findings

Identify usability issues and prioritize improvements.

---

## Quality Gates

Verify:

□ Representative users selected.

□ Critical tasks evaluated.

□ Metrics defined.

□ Results documented.

---

## Outputs

Generate:

- Usability Evaluation Report
- Findings Summary
- Improvement Recommendations
