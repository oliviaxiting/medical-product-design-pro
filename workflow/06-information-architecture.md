# Information Architecture Workflow

## Purpose

This workflow defines how AI should organize information within medical software interfaces.

The objective is to create a logical, efficient, and clinically meaningful information structure that supports rapid decision-making.

Information architecture should reduce cognitive workload and improve information discoverability.

---

## When to Use

Use this workflow whenever:

- Designing a new interface.
- Creating a dashboard.
- Adding new pages.
- Reorganizing navigation.
- Improving usability.

---

## Inputs

Required inputs include:

- Requirement Analysis
- Clinical Workflow
- Task Analysis
- PRD

---

## Process

### Step 1 — Identify Information Objects

List all information required by users.

Examples:

- Patient
- Device
- Alarm
- Trend
- Parameter
- Event
- Status

---

### Step 2 — Group Information

Organize related information together.

Avoid unnecessary fragmentation.

---

### Step 3 — Prioritize Information

Classify information according to clinical importance.

Priority should be based on:

- Patient safety
- Decision urgency
- Frequency of use

---

### Step 4 — Define Navigation

Design clear relationships between:

- Pages
- Modules
- Screens
- Functions

Navigation should minimize unnecessary steps.

---

### Step 5 — Validate Hierarchy

Confirm that users can easily locate critical information.

Reduce cognitive workload wherever possible.

---

## Quality Gates

Verify:

□ Information grouped logically.

□ Navigation supports workflow.

□ Critical information is prominent.

□ Information hierarchy is clear.

---

## Outputs

Generate:

- Site Map
- Navigation Structure
- Page Hierarchy
- Information Hierarchy
