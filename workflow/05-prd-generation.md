# PRD Generation Workflow

## Purpose

This workflow defines how AI should generate a Product Requirements Document (PRD) for medical software.

The objective is to transform analyzed requirements into a complete, implementable product specification while ensuring usability, safety, and regulatory compliance.

The generated PRD should be suitable for product managers, UX designers, developers, QA engineers, and regulatory teams.

---

## When to Use

Use this workflow whenever:

- A new product is planned.
- A new feature is designed.
- Existing functionality changes.
- A formal PRD is required.

---

## Inputs

Required inputs include:

- Requirement Analysis
- Clinical Workflow Analysis
- Task Analysis
- Risk Analysis

Reference:

- IEC 62366-1
- ISO 14971
- ANSI/AAMI HE75

---

## Process

### Step 1 — Review Inputs

Verify that all previous workflows have been completed.

Do not begin PRD generation if critical information is missing.

---

### Step 2 — Define Product Objectives

Describe:

- Business objective
- Clinical objective
- User objective

---

### Step 3 — Define Functional Requirements

For every feature specify:

- Purpose
- Inputs
- Outputs
- Business rules
- Constraints

---

### Step 4 — Define Information Architecture

Identify:

- Pages
- Navigation
- Information hierarchy
- Screen relationships

---

### Step 5 — Define Interaction Requirements

Specify:

- User flow
- System feedback
- State transitions
- Error handling
- Recovery mechanisms

---

### Step 6 — Verify Safety

Confirm that:

- Critical tasks are supported.
- Use errors are minimized.
- Risk controls are implemented.
- Human factors principles are applied.

---

### Step 7 — Generate PRD

Produce a complete PRD including:

- Background
- Objectives
- Scope
- User Profiles
- Clinical Workflow
- Functional Requirements
- Page List
- Interaction Specification
- Exception Handling
- Acceptance Criteria

---

## Quality Gates

Verify:

□ Requirements complete.

□ Clinical workflow validated.

□ Tasks analyzed.

□ Risks controlled.

□ Information architecture complete.

□ Interaction defined.

□ Acceptance criteria included.

---

## Outputs

Generate:

- Product Requirements Document (PRD)
