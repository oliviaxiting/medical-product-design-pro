# Usability Evaluation Output Specification

## Purpose

This document defines how AI should generate usability evaluation reports for medical software.

The objective is to objectively evaluate whether intended users can safely and effectively complete representative clinical tasks.

Evaluation should focus on measurable performance rather than subjective preference.

---

## Output Requirements

The report should evaluate representative users performing representative tasks under representative clinical conditions.

The report should identify usability strengths, weaknesses, and improvement opportunities.

---

## Required Structure

The report should include:

- Evaluation Objective
- Intended Users
- Clinical Environment
- Evaluation Tasks
- Success Criteria
- Findings
- Use Errors
- Recommendations
- Overall Assessment

Include only sections relevant to the evaluation.

---

## Evaluation Metrics

Whenever applicable include:

- Task Success Rate
- Task Completion Time
- Number of Use Errors
- Number of Recovery Attempts
- User Understanding
- Workflow Efficiency

Metrics should be measurable whenever possible.

---

## Writing Rules

Describe findings objectively.

Separate observations from recommendations.

Avoid unsupported conclusions.

Every recommendation should clearly explain why the improvement is needed.

---

## Medical Design Rules

Evaluation should consider:

- Clinical workflow
- Human factors
- Cognitive workload
- Information hierarchy
- Error prevention
- Patient safety
- Risk controls

---

## Quality Checklist

Before generating the report verify:

□ Representative users identified.

□ Representative tasks evaluated.

□ Findings supported by observations.

□ Recommendations are actionable.

□ Safety considerations included.

□ Improvement priorities identified.

Only output the report after all checks pass.
