# User Profile

## Purpose

This document defines how to identify and analyze intended users according to IEC 62366-1.

The objective is to ensure that interface design reflects the knowledge, responsibilities, experience, and limitations of different user groups.

User profiles should be established before workflow or interface design begins.

---

## Human Factors Principle

Different users perform different tasks under different conditions.

A single interface should not assume that all users have the same expertise, authority, or goals.

Interface decisions should always be based on the characteristics of the intended user.

---

## User Attributes

Each user profile should include:

- Clinical role
- Responsibilities
- Professional knowledge
- Technical experience
- Frequency of use
- Training level
- Decision authority
- Physical limitations
- Cognitive workload

---

## Typical Users

### Nurse

Primary responsibilities:

- Continuous monitoring
- Alarm response
- Bedside operation
- Clinical documentation

Characteristics:

- Frequent interruptions
- Time pressure
- High workload
- Rapid task switching

---

### Physician

Primary responsibilities:

- Clinical assessment
- Treatment decisions
- Trend analysis
- Therapy adjustment

Characteristics:

- Decision-oriented
- Less repetitive interaction
- Focus on clinical interpretation

---

### Biomedical Engineer

Primary responsibilities:

- Device configuration
- Calibration
- Maintenance
- Troubleshooting

Characteristics:

- High technical expertise
- Infrequent clinical operation

---

## Console Application

Console users generally require:

- High information density
- Multi-patient awareness
- Efficient navigation
- Keyboard and mouse support

---

## PAD Application

PAD users generally require:

- Touch interaction
- Bedside efficiency
- Simplified workflows
- Rapid confirmation

---

## AI Thinking Rules

Before generating any interface, identify:

- Who is the primary user?
- Who is the secondary user?
- What decisions does each user make?
- What information does each user require?
- What errors are most likely?

Never design for a generic "medical user."
