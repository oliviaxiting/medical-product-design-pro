# Hazard-Related Use Scenarios

## Purpose

This document defines how to identify hazard-related use scenarios according to IEC 62366-1.

The objective is to understand how normal user interactions may contribute to hazardous situations.

Hazard-related use scenarios form the connection between usability engineering and risk management.

---

## Human Factors Principle

Not every use error causes patient harm.

A hazard-related use scenario exists when a user action, combined with system conditions, could result in an unacceptable level of risk.

The interface should reduce both the probability and the severity of these situations.

---

## Hazard Analysis Process

For each workflow, identify:

Clinical Task

↓

Possible Use Error

↓

Hazardous Situation

↓

Potential Harm

↓

Risk Control

---

## Example

Clinical Task

Adjust infusion parameter

Possible Use Error

Incorrect parameter selected

Hazardous Situation

Incorrect infusion setting

Potential Harm

Patient receives inappropriate therapy

Risk Control

Confirmation

Visual verification

Parameter highlighting

Clinical review

---

## Console Application

Evaluate scenarios involving:

- Multiple patients
- Alarm acknowledgement
- Device configuration
- Historical review
- Clinical documentation

---

## PAD Application

Evaluate scenarios involving:

- Bedside confirmation
- Device adjustment
- Touch interaction
- Rapid intervention

---

## AI Thinking Rules

For every workflow ask:

Could this action contribute to patient harm?

If yes,

What interface improvement would reduce the risk?

Never evaluate interaction without considering clinical consequences.
