# Use Error

## Purpose

This document defines how to identify, analyze, and reduce use errors according to IEC 62366-1.

The objective is to prevent user actions that may result in hazardous situations, patient harm, or device misuse.

Use error analysis should be performed before interface implementation.

---

## Human Factors Principle

A use error is an action or lack of action that leads to an unintended result.

Use errors are not necessarily caused by user incompetence.

Many use errors originate from poor interface design, unclear workflows, insufficient feedback, or inappropriate information presentation.

The objective of usability engineering is to reduce the probability of use errors.

---

## Typical Sources of Use Error

Use errors commonly result from:

- Poor interface hierarchy
- Ambiguous terminology
- Similar-looking controls
- Hidden system status
- Inadequate feedback
- Poor navigation
- High cognitive workload
- Time pressure
- Environmental distractions

---

## Typical Medical Use Errors

Examples include:

- Selecting the wrong patient
- Confirming the wrong alarm
- Entering incorrect values
- Misinterpreting trends
- Overlooking critical alarms
- Adjusting the wrong parameter
- Cancelling the wrong operation
- Ignoring warning messages

---

## Console Application

Common risks include:

- Wrong patient selection
- Alarm overload
- Incorrect configuration
- Trend interpretation errors
- Monitoring interruptions

---

## PAD Application

Common risks include:

- Touching the wrong control
- Bedside confirmation errors
- Parameter adjustment mistakes
- Accidental operation
- Interrupted workflows

---

## AI Thinking Rules

For every interaction, AI should ask:

What mistakes could users make?

Why would they make them?

Can the interface prevent the error?

Can the interface detect the error?

Can the interface recover from the error?

Design should eliminate preventable use errors whenever possible.
