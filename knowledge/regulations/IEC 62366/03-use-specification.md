# Use Specification

## Purpose

This document defines how the intended use of a medical product should be described before interface design begins.

A complete use specification establishes the foundation for usability engineering.

Without a clear use specification, interface evaluation cannot be performed reliably.

---

## Objective

The objective is to describe the context in which the medical product will be used.

The interface should always be designed for a specific use context rather than a generic user.

---

## Required Elements

A complete use specification should define the following:

### Intended Medical Purpose

What clinical problem does the product solve?

---

### Intended Users

Examples include:

- Nurse
- Physician
- Biomedical Engineer
- Clinical Technician

Different users have different knowledge, responsibilities, and workflows.

---

### Patient Population

Examples include:

- Adult
- Pediatric
- Neonatal

Patient characteristics influence interface requirements.

---

### Use Environment

Examples include:

- Intensive Care Unit (ICU)
- Operating Room
- Emergency Department
- General Ward
- Transport Environment

Environmental conditions influence usability.

---

### Operating Conditions

Consider:

- Lighting
- Noise
- Time pressure
- Personal protective equipment
- Medical gloves
- Interruptions
- Emergency situations

The interface should remain usable under all expected conditions.

---

### Platform

Specify the interaction platform.

Examples:

- Console
- PAD

Interaction methods differ significantly between platforms.

---

## Console Considerations

Console products typically support:

- Long-duration monitoring
- Multi-patient management
- Detailed configuration
- Clinical documentation

Interfaces should prioritize efficiency and situational awareness.

---

## PAD Considerations

PAD products typically support:

- Bedside interaction
- Rapid confirmation
- Touch operation
- Patient-focused workflows

Interfaces should prioritize simplicity and speed.

---

## AI Thinking Rules

Before generating any design, AI should answer:

Who is using the product?

Where is it used?

Why is it being used?

What clinical task is being performed?

What environmental constraints affect interaction?

If any answer is unknown, the use specification is incomplete.
