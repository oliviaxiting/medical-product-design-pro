# Data Entry Review

## Purpose

This document defines review rules for clinical data entry based on ANSI/AAMI HE75 and IEC 62366-1.

The objective is to improve input accuracy, reduce documentation burden, and minimize use errors.

---

## Human Factors Principle

Data entry should require the minimum effort necessary while maintaining data accuracy and patient safety.

---

## HE75 Recommendations

### Minimize Input

Reduce manual typing whenever possible.

Prefer:

- Selection
- Search
- Scan
- Auto-complete

when clinically appropriate.

---

### Error Prevention

Prevent invalid input before submission.

Guide users rather than report errors afterward.

---

### Input Validation

Provide immediate and understandable validation.

Explain how to correct errors.

---

### Workflow

Arrange input fields according to clinical workflow.

Avoid unnecessary context switching.

---

### Patient Safety

Critical patient information should always be verified before submission.

Support confirmation for high-risk actions only.

---

## Console Application

Recommendations:

- Support keyboard shortcuts.
- Optimize repetitive documentation.
- Support barcode or device integration where appropriate.

---

## PAD Application

Recommendations:

- Minimize typing.
- Prefer touch-friendly controls.
- Support bedside documentation.

---

## Common Risks

- Excessive typing.
- Input errors.
- Poor validation.
- Repetitive documentation.
- Incorrect patient selection.

---

## Design Review Checklist

Before approving the design, verify:

□ Manual input is minimized.

□ Validation is immediate.

□ Errors are easy to correct.

□ Workflow follows clinical practice.

□ Console supports efficient documentation.

□ PAD supports bedside data entry.

---

## AI Review Rules

Always evaluate:

- Input efficiency
- Error prevention
- Clinical workflow
- Patient safety

Never increase documentation workload without clinical benefit.
