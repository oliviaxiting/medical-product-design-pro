# Color Review

## Purpose

This document defines color usage review rules based on ANSI/AAMI HE75 human factors guidance.

The objective is to improve recognition, reduce interpretation errors, and support safe clinical operation.

This document reviews functional color usage rather than defining brand colors.

---

## Human Factors Principle

Color should reinforce information, not replace it.

Users should always understand information without relying solely on color.

---

## HE75 Recommendations

### Status Indication

Use color consistently for system status.

Recommended examples:

- Normal
- Warning
- Critical
- Disabled
- Information

Status meaning should remain consistent throughout the product.

---

### Color Coding

Never use color as the only communication method.

Combine color with:

- Text
- Icons
- Labels
- Shape

---

### Contrast

Maintain sufficient contrast.

Avoid low-contrast combinations.

---

### Number of Colors

Limit functional colors.

Too many colors increase cognitive workload.

---

### Consistency

The same color should always represent the same meaning.

Avoid changing color meanings between pages.

---

## Console Application

Recommendations:

- Maintain clear status visualization.
- Avoid excessive color usage on dense dashboards.
- Highlight only clinically important information.

---

## PAD Application

Recommendations:

- Use stronger contrast.
- Support recognition under different lighting conditions.
- Reduce unnecessary visual decoration.

---

## Common Risks

- Color-only communication.
- Low contrast.
- Excessive accent colors.
- Inconsistent color meaning.
- Decorative color usage.

---

## Design Review Checklist

Before approving the design, verify:

□ Status colors are consistent.

□ Color is not the only indicator.

□ Contrast is sufficient.

□ Critical information is visually distinguishable.

□ Functional colors are limited.

---

## AI Review Rules

Always evaluate:

- Contrast
- Status consistency
- Color dependency
- Clinical readability

Never recommend color-only communication for safety-critical information.
