# Navigation Review

## Purpose

This document defines navigation review rules based on ANSI/AAMI HE75.

The objective is to support efficient clinical workflows while reducing cognitive workload and navigation errors.

---

## Human Factors Principle

Navigation should match users' mental models and clinical workflows.

Users should always know:

- Where they are.
- How they arrived.
- How to return.
- What actions are available.

---

## HE75 Recommendations

### Consistency

Navigation structure should remain consistent across the product.

Avoid changing menu locations between pages.

---

### Predictability

Navigation should behave as users expect.

Avoid unexpected page transitions.

---

### Workflow Support

Navigation should follow clinical tasks rather than technical architecture.

Frequently used functions should require fewer interactions.

---

### Visibility

Current location should always be obvious.

Highlight active pages.

---

### Return Path

Users should easily return to previous workflows.

Avoid losing task context.

---

### Navigation Depth

Avoid deep navigation structures.

Keep important functions within a small number of interactions.

---

## Console Application

Recommendations:

- Support persistent navigation.
- Keep monitoring available while navigating.
- Support multitasking.
- Maintain stable page layout.

---

## PAD Application

Recommendations:

- Reduce navigation depth.
- Prioritize current patient.
- Minimize context switching.
- Support rapid task completion.

---

## Common Risks

- Deep navigation.
- Hidden menus.
- Lost context.
- Excessive page switching.
- Inconsistent navigation.

---

## Design Review Checklist

Before approving the design, verify:

□ Navigation is predictable.

□ Current location is visible.

□ Return path is clear.

□ Workflow is efficient.

□ Console supports multitasking.

□ PAD supports bedside efficiency.

---

## AI Review Rules

Always evaluate:

- Workflow efficiency
- Navigation depth
- Orientation
- Task continuity

Never recommend navigation structures that interrupt clinical workflows.
