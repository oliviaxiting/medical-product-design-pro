# Icons Review

## Purpose

This document defines review rules for icon usage based on ANSI/AAMI HE75 human factors principles.

The objective is to improve recognition, reduce ambiguity, and support rapid understanding.

---

## Human Factors Principle

Icons should reinforce meaning rather than replace language.

Users should immediately recognize the purpose of an icon without unnecessary interpretation.

---

## HE75 Recommendations

### Recognition

Use familiar icons whenever possible.

Avoid inventing new symbols for common actions.

---

### Consistency

The same icon should always represent the same function.

Do not reuse icons for different meanings.

---

### Text Support

Icons should be accompanied by text when the meaning is safety-critical or potentially ambiguous.

Never rely solely on icons.

---

### Visual Simplicity

Keep icons simple.

Avoid unnecessary detail.

---

### Status Icons

Status icons should clearly distinguish:

- Normal
- Warning
- Critical
- Offline
- Disabled

---

## Console Application

Recommendations:

- Support rapid scanning.
- Use icons to reinforce table and dashboard information.

---

## PAD Application

Recommendations:

- Use larger icons.
- Increase spacing around icons.
- Maintain clear touch recognition.

---

## Common Risks

- Ambiguous icons.
- Decorative icons.
- Missing labels.
- Inconsistent meaning.
- Excessive icon usage.

---

## Design Review Checklist

Before approving the design, verify:

□ Icons are recognizable.

□ Icons support text.

□ Meaning is consistent.

□ Safety-critical icons include labels.

□ Icons improve scanning.

---

## AI Review Rules

Always evaluate:

- Recognition
- Consistency
- Semantic clarity
- Clinical safety

Never recommend icon-only communication for critical information.
